# 本地文件操作安全边界

Skill 管理器的“安全文件操作”面向个人用户正常管理本机文件的场景。

首版提供以下保护：

- ZIP 在应用暂存目录中解包，拒绝路径穿越、绝对路径、重复路径和越界符号链接；
- 计划和执行时都会检查 Skill 根目录及目标父链，拒绝静态符号链接逃逸；
- 父级符号链接下的 Skill 不允许移动或删除，避免破坏共享真实目录；
- 覆盖、移动和撤销使用计划指纹、备份与崩溃恢复；
- 删除进入 macOS 系统废纸篓，应用仅记录结果，不替代访达的恢复能力。

首版不把另一个拥有同等本机权限的恶意进程纳入威胁模型。若其他进程在一次操作的多个文件系统调用之间并发重命名目录、替换符号链接或篡改应用数据库，首版不承诺阻止所有 TOCTOU 攻击。执行文件操作时，请不要同时用其他工具重排相关 Skill 根目录。

后续若要支持不受信任的本机并发进程，应将复制、覆盖、移动、撤销和清理迁移到 capability-relative 文件 API，并为系统废纸篓补充平台级目录描述符实现。
