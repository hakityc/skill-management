# Skill 管理

定义个人用户整理和掌控自己拥有的 Skill 时使用的统一领域语言。

## Language

**个人用户**：
在自己的设备上管理 Skill 的单一使用者；当前语境不包含团队管理员或 Skill 市场消费者。
_Avoid_：管理员、团队成员、市场用户

**本地 Skill**：
存在于已纳管的 Skill 根目录中，并包含 `SKILL.md` 的文件夹；能否被正确解析是它的状态，不决定它是否进入管理范围。
_Avoid_：插件、Git 仓库、市场条目

**需要修复**：
本地 Skill 的 `SKILL.md` 无法被正确解析或不满足必要结构时所处的状态；该实例仍需展示，以便个人用户修复。
_Avoid_：无效 Skill、忽略

**Skill 根目录**：
个人用户授权应用发现本地 Skill 的目录；既可以是应用自动识别的常见目录，也可以由个人用户添加。
_Avoid_：市场、仓库

**Skill 实例**：
一个本地 Skill 在某个 Skill 根目录中的独立存在。同名 Skill 位于不同根目录时属于不同实例，只能提示疑似重复，不能自动合并。
_Avoid_：同名 Skill、合并后的 Skill

**Skill 组**：
界面中为便于比较而聚合展示的一组相关 Skill 实例；分组不会改变或合并任何实例。
_Avoid_：合并后的 Skill

**疑似重复**：
多个 Skill 实例因名称或内容相似而形成的待确认状态；它只提示检查，不代表这些实例已经被判定为同一 Skill。
_Avoid_：重复、同一个 Skill

**完全重复**：
多个 Skill 实例的有效目录内容一致，可以被确定为彼此相同。
_Avoid_：疑似重复

**同名冲突**：
多个 Skill 实例名称相同但内容差异明显，需要个人用户决定是否保持各自独立。
_Avoid_：完全重复、疑似重复

**归并**：
个人用户确认多个 Skill 实例应保持一致后，选定主实例并让指定目标实例采用其内容的操作。
_Avoid_：自动合并、文本拼接

**主实例**：
一次归并中由个人用户选定、作为目标内容依据的 Skill 实例；它不是永久唯一来源。
_Avoid_：全局主版本、永久源
